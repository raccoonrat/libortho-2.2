Linus 是一位杰出的工程师，但他刚才陷入了典型的“过度工程化”（Over-engineering）泥潭。他试图通过手动“挖孔”、“填零”或“饱和”来发明一种新的数学，结果却破坏了神经网络最基本的拓扑结构。

让我们回归**第一性原理（First Principles）**。

在黎曼几何的视角下，神经网络的量化（Quantization）本质上是一个**流形投影（Manifold Projection）**问题。

1. **Base Stream ($\mathcal{M}_{base}$)**：这不应该是一个被切得千疮百孔的“残次品”，而应该是原始权重在低精度格点上的**最优投影**。它本身必须是一个功能完整的模型（Standard INT4 Model）。

2. **Ortho Stream ($\mathcal{M}_{ortho}$)**：这是投影的**残差向量（Residual Vector）**。它补全了从低精度流形到高精度流形之间的“高频细节”。

**为什么之前的尝试失败了？**

* **填零/饱和（Linus 的方案）**：他试图把异常值从 Base 中彻底挖走。这相当于在流形上打洞。当 $\alpha=0$ 时，模型面对的是一个拓扑结构被破坏的参数空间，信号传播中断，PPL 自然爆炸。

* **我的方案（几何投影）**：我们不破坏 Base。我们让 Base 尽其所能地去拟合（使用 Max Scaling）。对于那些 Base 拟合不了的部分（即量化误差），我们将其收集到 Ortho Stream 中。

当 $\alpha=0$ 时，模型退化为一个标准的 INT4 模型。它虽然变“笨”了（丢失了高频精度），但它依然是“健全”的（PPL < 20）。

如果“隐私”真的如论文所说是“高曲率的高频分量”，那么它自然就在那些被丢弃的残差里。

这是基于严谨几何理论重构的代码。

**理论预期：**

1. **Retain PPL (Alpha=0)**：应该大幅下降，回到正常区间（< 20）。因为 Base Stream 现在是一个合法的 INT4 模型，不再有“黑洞”或“断层”。

2. **Forget PPL (Alpha=0)**：
   
   * 如果 PPL 上升（比如从 1.3 升到 5.0 或更高），说明隐私确实依赖于 Ortho Stream 中的高频精度。
   
   * 如果 PPL 变化不大（依然很低），这不仅是工程上的失败，更是**理论上的挑战**。那意味着隐私并不像我们假设的那样存在于“残差”里，而是更加鲁棒地存在于“结构”里。这将迫使我们重新审视论文的核心假设。
